# PR-12.1B-H — Capture Health & Data Quality Audit

## Summary

Adds a read-only audit that evaluates forward live quote capture artifacts (or M12.1C spike dirs) and emits a research-readiness verdict with continuity, spread, and BTC spot join metrics.

## CLI

```bash
npm run research:capture-health-audit -- --capture-run-dir data/live-capture/kalshi-ws-spike/<runId>
```

Outputs:

- `data/research-results/capture-health-audit.json`
- `data/reports/capture-health-audit.html`

## Verdicts

- `capture-empty`
- `capture-invalid`
- `capture-too-short`
- `capture-gappy`
- `capture-no-btc-spot`
- `capture-zero-spread-suspicious`
- `capture-research-ready`

Thresholds are configurable via CLI flags (e.g. `--min-duration-seconds`, `--max-p90-gap-ms`).

## Quality gates

```bash
npm run lint
npm run build
npm run test
npm run research:capture-health-audit -- --capture-run-dir data/live-capture/kalshi-ws-spike/<runId>
```
