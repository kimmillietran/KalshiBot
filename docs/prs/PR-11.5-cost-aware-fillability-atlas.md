# PR 11.5 — Cost-Aware + Fillability Atlas

## Why calibration gap is not equal to tradable edge

The mispricing atlas reports **signed calibration error** (`averageImpliedProbability − realizedFrequency`). That answers whether a bucket is miscalibrated on average, but it ignores:

- **Bid/ask spread** — you pay the ask (or receive the bid) when entering a fade.
- **Fees** — Kalshi schedule fees depend on execution price and contract count.
- **Fillability** — many replay steps may lack valid two-sided quotes or only appear in wide-spread regimes.

A bucket can show a large positive calibration gap while remaining **untradeable** after spread and fees, or while most observations sit in missing-quote cohorts.

This milestone adds **additive** cost-aware diagnostics. It does **not** change existing atlas fields, hypothesis generation, validation thresholds, replay, or strategy harness behavior.

## How spread-adjusted and fee-adjusted EV are computed

Per bucket (and per fillability cohort), we reuse the atlas calibration gap:

```
rawCalibrationGap = averageImpliedProbability − realizedFrequency
grossExpectedValueCents = rawCalibrationGap × 100
```

The gross value is interpreted as the expected fade edge per $1 binary contract before execution costs, using the same sign convention as the atlas (positive gap ⇒ overconfident YES market).

For observations with valid bid/ask:

```
averageHalfSpreadCents = mean((yesAskCents − yesBidCents) / 2)
spreadAdjustedExpectedValueCents = grossExpectedValueCents − averageHalfSpreadCents
```

Fees use the existing Kalshi taker fee schedule (`computeKalshiScheduleFeeCents`) at the fade execution price:

- Overconfident (fade YES) ⇒ buy NO at `100 − yesBidCents`
- Underconfident (fade NO) ⇒ buy YES at `yesAskCents`

```
averageFeeCents = mean(per-observation schedule fee)
feeAdjustedExpectedValueCents = spreadAdjustedExpectedValueCents − averageFeeCents
minimumRequiredEdgeCents = averageHalfSpreadCents + averageFeeCents
```

## Fillability cohort definitions

Spread tiers are deterministic and configurable via `DEFAULT_COST_AWARE_ATLAS_CONFIG`:

| Cohort | Rule |
|--------|------|
| `all` | Every observation in the bucket |
| `validBidAsk` | `quoteStatus === "valid"` |
| `tightSpread` | valid quote and spread ≤ 2% |
| `mediumSpread` | valid quote and 2% < spread ≤ 5% |
| `wideSpread` | valid quote and spread > 5% |
| `missingOrInvalidQuote` | missing, one-sided, or invalid quotes |

Spread percent uses the YES side: `(yesAsk − yesBid) / yesAsk × 100`, matching `marketFeatures.spreadPercent`.

The **primary cohort** for tradeability classification is `validBidAsk`, falling back to `all` when needed.

## Tradeability classifications

| Class | Meaning |
|-------|---------|
| `tradeable-positive` | Fee-adjusted EV > neutral threshold with adequate quotes |
| `tradeable-negative` | Fee-adjusted EV < −neutral threshold |
| `gross-only` | Gross EV positive but fee-adjusted EV ≤ 0 |
| `untradeable-wide-spread` | Majority wide-spread quotes and non-positive fee-adjusted EV |
| `untradeable-missing-quotes` | No valid quotes or majority missing/invalid |
| `underpowered` | Observations < `minSampleThreshold` (default 30) |
| `unknown` | Insufficient data to classify |

## What this milestone does not yet prove

- **Not a strategy** — no orders, sizing, or portfolio simulation.
- **Not causal** — bucket EV is descriptive, not out-of-sample alpha.
- **Not execution realism** — no partial fills, queue position, or maker rebates.
- **Not hypothesis promotion** — validation thresholds and candidate generation are unchanged.
- **Not a guarantee that promising hypotheses survive** — the report highlights where gross edges collapse after cost; downstream milestones must re-test candidates under execution constraints.
