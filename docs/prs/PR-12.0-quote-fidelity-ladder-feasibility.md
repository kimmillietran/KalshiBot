# PR 12.0 — Quote Fidelity & Ladder Feasibility Gate

## Why this exists

Pre-M12 recon found that the current KXBTC15M historical corpus cannot support executable cross-strike no-arb research:

- One market per `event_ticker` (no multi-strike ladder)
- Historical Kalshi quotes are live-close-only proxies
- Registry shows ~100% zero spread
- Real executable bid/ask and displayed depth are missing
- `eventTicker` is not persisted in silver `MarketWindow`

M12.0 adds an honest feasibility gate before building any cross-strike scanner.

## What this gate answers

```text
Can the current KXBTC15M historical corpus support executable static no-arb research?
```

A **blocked** verdict is a valid success. The goal is to prevent fake arb research on data that cannot support it.

## Primary input

```text
data/research-datasets/KXBTC15M/dataset-registry.json
```

Secondary (sampled): bronze fixtures and research outputs for field availability checks.

**Not used as primary:** mispricing-atlas, hypothesis-trade-replay, cost-aware-atlas (downstream/aggregated).

## Verdicts

| Verdict | Meaning |
|---------|---------|
| `blocked-no-ladder` | No multi-strike events in corpus |
| `blocked-close-only-quotes` | Quotes are close-only / zero-spread synthesized |
| `blocked-needs-live-quotes` | No executable bid/ask |
| `blocked-missing-event-metadata` | Cannot recover event grouping |
| `blocked-insufficient-data` | Empty or unusable registry |
| `proceed-static-parity` | Single-market parity research may be feasible |
| `proceed-cross-strike-ladder` | Multi-strike ladder + executable quotes |
