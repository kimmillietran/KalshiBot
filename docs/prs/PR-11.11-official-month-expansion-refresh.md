# PR 11.11 — Official Month Expansion / Calibration-Fade Evidence Refresh

## Why this exists after M11.10

M11.10 found that excluding derived-sensitive month 2025-12 leaves positive PnL (+2,302¢, 29% retention) but **Jan 2026 still dominates** remaining PnL at 84.7%. The recommendation was `collect-more-official-months`.

M11.11 asks whether **adding more official/non-derived historical coverage** broadens calibration-fade evidence enough to justify full M12 Trade-PnL OOS Overlay.

## What this milestone does

1. **Audits month coverage** — official vs derived-sensitive status, market/observation/replay fill counts, importable months
2. **Checks importability** — uses existing `research:coverage-plan` and `research:generate-expansion-import-config`
3. **Optionally imports** — `research:execute-expansion-import --execute` + `research:rebuild-after-expansion` (never a parallel importer)
4. **Optionally reruns evidence chain** — hypotheses through derived-month-pnl-sensitivity
5. **Compares before vs after** — PnL, month concentration, verdicts, recommendations

## Official vs derived status

| Status | Meaning |
|--------|---------|
| `official` | Calendar month is not in `DERIVED_SETTLEMENT_SENSITIVE_MONTHS` (default: 2025-12) |
| `derived-sensitive` | Known derived-settlement-sensitive month; excluded from official expansion targets |

Per-market derived settlement flags come from research dataset snapshots when discovered; month-level heuristics apply when per-trade flags are unavailable.

## Commands

### Audit + comparison (default)

```bash
npm run research:official-month-refresh
```

Refreshes coverage plan, assesses importability, compares current artifacts before/after (no import).

### With import + evidence chain rerun

```bash
npm run research:official-month-refresh -- --execute-import --rerun-evidence-chain --research-results-dir data/research-results
```

### Evidence chain (manual rerun after import)

```bash
npm run research:hypotheses
npm run research:hypothesis-validation
npm run research:cost-aware-atlas
npm run research:hypothesis-trade-replay
npm run research:oos-power-correction
npm run research:family-verdict

```

## Decision recommendations

| Recommendation | Meaning |
|----------------|---------|
| `proceed-to-trade-pnl-oos` | Evidence broadened; full M12 may be worth building |
| `collect-more-official-months` | Positive but still month-concentrated or underpowered |
| `pause-calibration-fade` | New data weakens or concentration remains severe |
| `pivot-new-family` | No more official months available; stop squeezing calibration-fade |
| `insufficient-new-data` | Import ran but added no meaningful coverage |
| `blocked-import-refresh` | Import or rebuild failed |

## What this does not prove

- Not validated strategy
- Not live-trading readiness
- Not statistically proven alpha

A good negative result (pivot or pause) is acceptable and actionable.

## What should happen next

If recommendation is `pivot-new-family`, audit alternative families:

- cross-strike no-arb
- spot-to-Kalshi lead-lag
- microstructure / quote staleness

If `collect-more-official-months`, run import with `--execute-import` when Kalshi API access is available.

If `proceed-to-trade-pnl-oos`, proceed to full M12 Trade-PnL OOS Overlay.
