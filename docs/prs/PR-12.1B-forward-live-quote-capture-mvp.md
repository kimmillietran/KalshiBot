# PR 12.1B — Forward Live Quote Capture MVP

## Why this exists

M12.0 blocked historical KXBTC15M for executable microstructure research. M12.1C/M12.1C.1 proved authenticated live Kalshi WebSocket capture works. M12.1B turns that spike into a duration-bounded forward capture MVP.

## What this builds

- Duration-bounded KXBTC15M forward capture (hours-capable)
- Auto market discovery + rollover for 15-minute contracts
- Append-only JSONL: raw WS, top-of-book, BTC spot, market metadata
- Sequence gap detection + resync snapshot requests
- Reconnect/resubscribe on disconnect
- Coinbase BTC spot sidecar (degraded, not fatal)
- Capture health JSON + HTML report

**Not included:** orders, strategy, scanner, vendor importer, database, daemon.

## CLI

Use direct `npx tsx` on Windows (npm arg forwarding can be unreliable):

```powershell
npx tsx scripts/live/runForwardQuoteCapture.ts --series KXBTC15M --duration-minutes 60 --max-markets 3 --capture-btc-spot --output-dir data/live-capture/forward-quotes
```

Dry-run (no credentials):

```powershell
npx tsx scripts/live/runForwardQuoteCapture.ts --dry-run --series KXBTC15M --duration-minutes 1 --max-markets 1 --capture-btc-spot
```

Live (credentials required):

```powershell
$env:KALSHI_API_KEY_ID="..."
$env:KALSHI_API_PRIVATE_KEY_PATH="C:\Users\Kimmillie\.kalshi\kalshi-api.key.txt"

npx tsx scripts/live/runForwardQuoteCapture.ts --series KXBTC15M --duration-minutes 5 --max-markets 1 --capture-btc-spot
```


Runtime artifacts are gitignored (`data/live-capture/`).

## Top-of-book ask derivation

YES asks / NO asks are derived from opposite-side best bids (Kalshi displayed book convention):

- `yesAsk = 100 - noBid`
- `noAsk = 100 - yesBid`
- Ask sizes use opposite-side bid size at implied ask level

## Verdicts

| Verdict | Meaning |
|---------|---------|
| `capture-mvp-success` | Live capture with valid books |
| `degraded-capture` | Captured with gaps/errors but partial data |
| `dry-run-ok` | Mock plumbing validated |
| `blocked-missing-credentials` | No Kalshi API credentials |
| `blocked-ws-auth` | WS connection/auth failed |
| `blocked-market-discovery` | No active markets found |
| `blocked-no-valid-books` | No snapshots / valid top-of-book |
