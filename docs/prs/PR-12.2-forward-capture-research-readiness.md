# PR 12.2-R — Forward Capture Research Readiness Gate

## Summary

Read-only gate that decides when accumulated forward-capture data is sufficient to start offline executable microstructure research (lead-lag, quote staleness, same-market parity, calibration-fade spread realism).

A successful 60s smoke proves plumbing only — not research readiness.

## CLI

```bash
npm run research:forward-capture-readiness
```

## Inputs

- `data/live-capture/forward-quotes/<runId>/`
- `data/live-capture/kalshi-ws-spike/<runId>/`

Each run directory should contain `capture-health.json` plus optional `top-of-book.jsonl`, `btc-spot.jsonl`, and `raw-messages.jsonl`.

## Outputs

- `data/research-results/forward-capture-readiness.json`
- `data/reports/forward-capture-readiness.html`

## Test plan

- [x] No runs → `not-ready-no-data`
- [x] Short smoke → `not-ready-too-short` / `keep-capturing`
- [x] Gappy runs, missing BTC, 24h lead-lag ready, parity ready
- [x] Aggregation by day and runId
- [x] Deterministic JSON/HTML + CLI smoke
