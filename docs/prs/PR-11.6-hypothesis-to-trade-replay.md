# PR-11.6 — Hypothesis-to-Trade Replay

## Summary

Closes the research seam between descriptive hypothesis validation and executable simulation by replaying each atlas hypothesis as a simple, cost-aware cross-spread trade rule and reporting net PnL evidence.

## Why hypothesis validation is not enough

Hypothesis validation (M9.x) scores calibration robustness: time stability, regime stability, concentration, and leave-one-period-out behavior. A hypothesis can pass those checks while still being untradeable (wide spreads, missing quotes) or unprofitable after crossing the spread and paying fees. This milestone adds a **faithful dumb replay** — not promotion, not optimization — to answer: “If we had traded this hypothesis literally, would we have made money?”

## Replay assumptions

- **Entry:** cross-spread (buy YES or NO at the available ask).
- **Exit:** hold to settlement; payout uses observed market outcome.
- **Fees:** Kalshi taker fee schedule by default.
- **Filters:** configurable max spread, min expected net edge, optional slippage buffer.
- **Skips:** missing/invalid quotes, wide spread, insufficient edge.
- **Bucket gating:** same atlas bucket matchers and refinement filters as validation.

## Limitations

- Cross-spread entry is conservative but still ignores queue position and adverse selection.
- No partial fills, no intraday exits, no portfolio constraints.
- Expected edge filter uses bucket-level calibration error applied per observation — a coarse proxy.
- Lead-lag and non-atlas hypotheses are reported as unsupported in M11.6.
- `cost-aware-atlas.json` is optional context only; replay uses step quotes from research outputs.

## Interpreting results

| Outcome | Meaning |
|---------|---------|
| Positive net replay | Descriptive edge may survive execution costs — still not promotion-ready. |
| Negative net replay with fills | Calibration gap existed descriptively but costs/spread consumed it. |
| Zero fills / killed by spread | Hypothesis is observational only under current liquidity assumptions. |
| Small trade count warnings | Treat PnL as indicative; dependence across days/markets remains. |

## CLI

```bash
npm run research:hypothesis-trade-replay
```

Useful flags:

- `--input data/research-results/hypothesis-candidates.json`
- `--atlas data/research-results/mispricing-atlas.json`
- `--cost-aware-atlas data/research-results/cost-aware-atlas.json`
- `--max-spread-cents 10`
- `--min-net-edge-cents 0`
- `--execution-mode cross-spread`
- `--official-only`

## Outputs

- `data/research-results/hypothesis-trade-replay.json`
- `data/reports/hypothesis-trade-replay.html`
