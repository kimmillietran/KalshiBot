# PR-12.1C.1 — Kalshi Auth Signing + Private-Key Path Support

## Summary

Makes the M12.1C capture spike capable of authenticated live Kalshi WebSocket connections by loading private keys from file path or PEM env vars, generating official RSA-PSS auth headers, and connecting via Node `ws` with handshake headers.

## Signing behavior

Kalshi signing string:

```text
${timestampMs}${method}${pathWithoutQuery}
```

WebSocket handshake uses:

```text
GET /trade-api/ws/v2
```

Headers:

- `KALSHI-ACCESS-KEY`
- `KALSHI-ACCESS-TIMESTAMP`
- `KALSHI-ACCESS-SIGNATURE` (RSA-PSS SHA-256, base64)

## Private key loading

Precedence:

1. `--private-key-path` CLI flag
2. `KALSHI_API_PRIVATE_KEY_PATH`
3. `KALSHI_API_PRIVATE_KEY`
4. `KALSHI_PRIVATE_KEY`

Supports Windows paths and escaped `\n` in raw PEM env vars.

## Security

Artifacts redact PEM contents, signature values, and private-key env key names. Health reports expose only safe metadata (`privateKeySource`, fingerprint prefix, `authHeadersGenerated`).
