# PR 11.3 — Research Observation Enrichment Bridge

## Summary

Introduces a canonical computed feature bag on mispricing observations so research modules can stop re-deriving scattered features independently.

## New module

`src/lib/data/research/mispricingAtlas/`

| File | Responsibility |
|------|----------------|
| `researchObservationFeaturesTypes.ts` | `ComputedResearchFeatures`, enrichment context |
| `enrichResearchObservationFeatures.ts` | `enrichResearchObservationFeatures()`, field mapper |

## Wiring

`parseMispricingObservations.buildObservation()` now calls the bridge and attaches `computedFeatures` while preserving legacy top-level fields.

## Constraints

- Canonical computations only (`percentToTarget`, `estimateRealizedVolatility`, `computeResearchObservationMomentumPercent`, temporal extractors)
- No DAG engine, no new indicators (EMA/ATR/VWAP/RSI)
- Does not read `TradeDecision.features` (engine 5-bar momentum / stdDev differ from research metrics)
- Additive only — `mispricing-atlas.json` schema unchanged

## Before / after

| Concern | Before | After |
|---------|--------|-------|
| Feature source | Inline in `buildObservation` | `enrichResearchObservationFeatures()` |
| Stable feature bag | None | `observation.computedFeatures` |
| Top-level fields | Same | Same values (mirrored from bag) |
| Atlas / validation JSON | Unchanged | Unchanged |
