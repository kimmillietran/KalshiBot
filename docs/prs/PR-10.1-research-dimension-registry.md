# PR 10.1 — Research Dimension Registry

## Summary

Introduces a centralized research dimension registry and refactors atlas accumulation, bucket matching, hypothesis generation, and validation filtering to consume it — with **no behavior change**.

## New module

`src/lib/data/research/dimensions/`

| File | Responsibility |
|------|----------------|
| `types.ts` | `ResearchDimension`, `ResearchAxisGroup`, dimension IDs |
| `bucketDefinitions.ts` | Probability, time, moneyness, volatility bucket defs |
| `extractors.ts` | Observation → dimension value |
| `matchers.ts` | Single/multi-axis bucket matching |
| `registry.ts` | Dimension + axis group registry, composite templates |
| `index.ts` | Public barrel |

## Refactored consumers

- `mispricingAtlasIncrementalAccumulator.ts` — registry-driven Cartesian accumulation
- `matchMultiAxisBucket.ts` — re-exports from dimensions matchers
- `mispricingAtlasBuckets.ts` — re-exports from dimensions bucketDefinitions
- `filterObservationsForAtlasBucket.ts` — uses `observationMatchesResearchAxisGroupBucket`
- `observationMatchesAtlasBucket.ts` — delegates to registry
- `buildHypothesisCandidates.ts` — `collectAtlasBucketGroupsFromNormalizedAtlas`
- `resolveHypothesisBucketThresholds.ts` — `resolveAxisGroupSampleThreshold`

## Behavior

JSON schemas and pipeline outputs unchanged. `probabilityRegime` remains a special axis group (regime tags, not a core dimension).
