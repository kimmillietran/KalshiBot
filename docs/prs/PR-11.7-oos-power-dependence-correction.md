# PR 11.7 — Out-of-Sample, Power, and Dependence-Aware Correction

## Summary

Adds a stricter statistical overlay for hypothesis evaluation: temporal train/validation/holdout splits, power/MDE diagnostics, and Benjamini-Yekutieli FDR correction under arbitrary dependence.

## Why this matters

- **Raw observations overstate sample size** when multiple steps come from the same market-day block.
- **Overlapping atlas buckets** test correlated hypotheses; simple BH FDR can be anti-conservative.
- **In-sample bucket selection** inflates apparent edge unless holdout months are reserved.
- **MDE** clarifies whether observed edge is distinguishable from noise at realistic power.

## What this milestone does

- Deterministic month splits (explicit CLI or 60/20/20 default)
- Per-candidate holdout p-values, BY q-values, MDE, effective sample size
- `finalStatisticalVerdict`: pass | fail | underpowered | insufficient-data | skipped
- JSON + HTML artifacts

## What it does not do

- Does not change hypothesis generation defaults
- Does not auto-promote candidates
- Does not replace existing robustness scoring
- Block-bootstrap reality check is scaffolded; BY is the active correction path

## CLI

```bash
npm run research:oos-power-correction
```

Outputs:

- `data/research-results/oos-power-correction.json`
- `data/reports/oos-power-correction.html`
