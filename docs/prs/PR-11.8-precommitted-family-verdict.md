# PR 11.8 — Precommitted Calibration-Fade Family Verdict

## Why this exists

M11.6 showed positive in-sample net replay under spread + taker fees, but that is **not** alpha proof. M11.7 adds holdout calibration-edge testing, MDE, and dependence-aware correction — a separate evidence layer from trade-replay PnL.

M11.8 produces a **precommitted family verdict** that joins cost-aware atlas, trade replay, OOS/power/correction, and optional derived sensitivity into one promotion decision.

## CLI

```bash
npm run research:family-verdict
```

## Outputs

- `data/research-results/calibration-fade-family-verdict.json`
- `data/reports/calibration-fade-family-verdict.html`

## Precommitted gates

A hypothesis promotes only when **all** gates pass:

1. Positive net replay PnL (M11.6)
2. Fillability thresholds using markets/days — not raw filled count
3. Positive holdout calibration edge (M11.7)
4. Clears MDE and is not underpowered (M11.7)
5. Passes dependence-aware correction (M11.7)
6. Derived sensitivity acceptable when artifact present; unknown when absent

## Verdict semantics

| Family verdict | Meaning |
|----------------|---------|
| `promote-family` | At least one hypothesis passes all gates |
| `continue-research` | Positive but incomplete evidence |
| `reject-family` | Sufficient evidence, all fail decisively |
| `underpowered` | M11.7 marks candidates underpowered — not proof of no edge |
| `insufficient-data` | Too little fillable/OOS data |
| `blocked-by-missing-artifacts` | Required upstream JSON missing |

## Important caveats

- `1,163 filled trades` ≠ `1,163 independent bets`
- M11.7 tests holdout **calibration edges**, not M11.6 trade-replay PnL
- Promotion means eligible for execution realism / paper shadow — **not live trading**
