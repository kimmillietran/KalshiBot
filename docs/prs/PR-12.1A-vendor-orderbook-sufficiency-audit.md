# PR 12.1A — Third-Party Orderbook Sufficiency Audit

## Why this exists

M12.0 proved the current KXBTC15M historical corpus is blocked for executable microstructure research:

- 8,138 markets, 8,138 single-strike events
- 100% live-close-only / zero-spread quotes
- Verdict: `blocked-no-ladder`

The post-M12 decision memo concluded third-party orderbook data may partially unblock research, but only if sample quality is proven. M12.1A makes that conclusion operational.

## What this audit answers

```text
Can vendor-provided Kalshi orderbook data support executable research for
KXBTC15M lead-lag / quote staleness / same-market parity, and possibly KXBTCD ladder research?
```

This is **not** a vendor importer, trading milestone, or strategy scanner.

## Sufficiency families

1. **KXBTC15M lead-lag / quote staleness** — sub-minute snapshots, sizes, non-zero spreads
2. **Same-market parity** — real YES/NO book with prices, quantities, timestamps
3. **KXBTC15M ladder** — product-blocked unless sample proves multi-strike events
4. **KXBTCD ladder** — requires multi-strike events, floor_strike, co-timestamped quotes
