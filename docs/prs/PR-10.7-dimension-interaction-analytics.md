# PR 10.7 — Dimension Interaction Analytics

## Summary

Read-only interaction quality analytics across composite research axis groups (e.g. Probability × Momentum, Probability × Hour). **Not SHAP. Not feature importance.**

## New module

`src/lib/data/research/dimensionInteractionAnalytics/`

| File | Responsibility |
|------|----------------|
| `dimensionInteractionAnalyticsTypes.ts` | Report shape, defaults, error codes |
| `computeInteractionMetrics.ts` | Entropy, sparsity, coverage, interaction score |
| `analyzeDimensionInteractions.ts` | Group candidates/validations, build rankings |
| `loadDimensionInteractionAnalyticsInputs.ts` | Load upstream JSON artifacts |
| `buildDimensionInteractionAnalyticsReport.ts` | Orchestrator + JSON serializer |
| `serializeDimensionInteractionAnalyticsHtml.ts` | Standalone HTML report |
| `index.ts` | Public barrel |

## CLI

```bash
npm run research:interaction-analysis
```

Outputs:

- `data/research-results/research-interaction-analysis.json`
- `data/reports/research-interaction-analysis.html`

## Metrics per composite group

- Candidate count, pass rate, average robustness
- Near-promising frequency (failure analysis or heuristic)
- Average calibration error, coverage quality
- Bucket sparsity, entropy, interaction score

## Rankings

- Best / weakest interactions (by interaction score)
- High-potential (near-promising, low pass rate, decent robustness)
- High-noise (high entropy + sparsity, low pass rate)

## Constraints

Read-only. No changes to validation, hypothesis generation, or synthesis.
